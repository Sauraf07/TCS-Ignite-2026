# 📖 Overview

The **TCS B.Sc. Ignite & Smart Hiring Assessment** is designed to evaluate a candidate's:

- Numerical Aptitude
- Verbal Ability
- Logical Reasoning
- Programming Skills (Optional Coding)

The assessment is conducted through the **TCS iON** platform at designated test centers.

> **Note:** The coding section is optional, but attempting it can improve your overall performance and may strengthen your chances for the Ignite profile. Always refer to the latest official notification for the hiring cycle you're applying to. :contentReference[oaicite:0]{index=0}

---

# ⏱ Assessment Pattern

| Section | Questions | Time |
|----------|----------:|-----:|
| Numerical Ability | 20 | 30 Minutes |
| Verbal Ability | 20 | 30 Minutes |
| Reasoning Ability | 20 | 30 Minutes |
| Coding Ability *(Optional)* | 1 Programming Question | 30 Minutes |
| **Total** | **61** | **120 Minutes** |

# 🔒 Section Navigation

The assessment follows **section-wise timing**.

This means:

- Complete one section before moving to the next.
- You cannot return to a previous section after its timer ends.
- Manage your time carefully within each section.

# ⏳ Time Management Strategy

### Numerical Ability

- Spend about **1–1.5 minutes** per question.
- Skip lengthy calculations initially and revisit if time permits.

### Verbal Ability

- Finish grammar and vocabulary questions first.
- Attempt reading comprehension afterward.

### Reasoning Ability

- Solve direct questions before tackling lengthy puzzles.

### Coding

- Read the problem completely.
- Understand input/output formats.
- Handle edge cases.
- Test with sample inputs before submitting.

---

# 🎯 Recommended Attempt Strategy

| Step | Action |
|------|--------|
| 1 | Read all instructions carefully. |
| 2 | Answer easy questions first. |
| 3 | Skip lengthy questions initially. |
| 4 | Keep track of the timer. |
| 5 | Attempt the coding question if you are comfortable with programming. |
| 6 | Review your answers if time permits. |

# ❌ Common Mistakes to Avoid

- Spending too much time on one question.
- Ignoring instructions.
- Leaving easy questions unanswered.
- Not reading coding constraints.
- Forgetting edge cases in programming.
- Panicking because of sectional timers.
- Rushing without reviewing answers.

# ❓ Frequently Asked Questions

### Is the coding section compulsory?

The coding section is listed as **optional** in the current assessment pattern. Attempting it is generally recommended if you have programming knowledge. :contentReference[oaicite:1]{index=1}

---

### Is there negative marking?

The official sample materials do not indicate negative marking for the integrated assessment. Always verify this in the latest instructions for your hiring cycle. :contentReference[oaicite:2]{index=2}

---

### Can I go back to a previous section?

No. The assessment uses **section-wise timing**, and once a section is completed, you cannot revisit it.

---

### Which programming language should I choose?

Choose the language you are most comfortable with. If you're already proficient in **Python**, it is an excellent option because of its simple syntax and rapid development.
